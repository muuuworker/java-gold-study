1. JDBCの概要

table JDBC主要クラス・インタフェース一覧

クラス / インタフェースパッケージ役割
DriverManagerjava.sqlJDBCドライバを管理し、Connection(接続)を取得する
Connectionjava.sqlデータベースへの接続を表す。トランザクションの管理もここ
Statementjava.sqlSQL文を実行する（SQLインジェクションに注意）
PreparedStatementjava.sqlプレースホルダ「?」付きSQLを事前コンパイル。安全かつ高速
CallableStatementjava.sqlストアドプロシージャを呼び出す
ResultSetjava.sqlSELECT文の実行結果を格納するカーソル
SQLExceptionjava.sqlDB操作で発生する検査例外

description JDBCの基本的な流れ

① 接続取得（DriverManager.getConnection）→ ② Statement作成 → ③ SQL実行 → ④ ResultSet処理 → ⑤ クローズ（try-with-resourcesで自動化）

2. 接続とStatement

table SQL実行メソッドの使い分け

メソッド用途戻り値
executeQuery(sql)SELECT文ResultSet
executeUpdate(sql)INSERT / UPDATE / DELETE / DDLint（更新件数）
execute(sql)どんなSQLでも実行boolean（ResultSetがあれば true）
// try-with-resourcesでConnection, Statementを自動クローズ
String url = "jdbc:h2:mem:testdb"; // H2インメモリDBの例

try (Connection conn = DriverManager.getConnection(url, "sa", "");
     Statement stmt = conn.createStatement()) {

    // DDL: テーブル作成
    stmt.execute("CREATE TABLE products (id INT PRIMARY KEY, name VARCHAR(50), price INT)");

    // DML: データ挿入 (executeUpdate: 更新件数を返す)
    int rows = stmt.executeUpdate("INSERT INTO products VALUES (1, 'Coffee', 500)");
    System.out.println("挿入件数: " + rows);

    // DQL: データ取得 (executeQuery: ResultSetを返す)
    ResultSet rs = stmt.executeQuery("SELECT * FROM products");
    while (rs.next()) {
        System.out.println(rs.getInt("id") + ": " + rs.getString("name") + " ¥" + rs.getInt("price"));
    }

} catch (SQLException e) {
    System.out.println("エラー: " + e.getMessage());
}
terminal Output 挿入件数: 1
1: Coffee ¥500

3. PreparedStatement（推奨）

table Statement vs PreparedStatement

StatementPreparedStatement
SQLの作り方文字列を直接連結「?」プレースホルダ使用
SQLインジェクション脆弱安全（エスケープ自動処理）
パフォーマンス毎回コンパイル事前コンパイル済み（高速）
パラメータ設定なし（文字列結合）setString(1, value)等

description プレースホルダ「?」のインデックスは1始まり

PreparedStatementのパラメータインデックスは 0ではなく1始まりです。試験でよく問われます。

String insertSql = "INSERT INTO products (id, name, price) VALUES (?, ?, ?)";
//                                                                  1   2      3

try (Connection conn = DriverManager.getConnection(url, "sa", "");
     PreparedStatement pstmt = conn.prepareStatement(insertSql)) {

    // 1件目
    pstmt.setInt(1, 2);
    pstmt.setString(2, "Tea");
    pstmt.setInt(3, 300);
    pstmt.executeUpdate();

    // パラメータをリセットして2件目 (clearParameters()は省略可)
    pstmt.setInt(1, 3);
    pstmt.setString(2, "Juice");
    pstmt.setInt(3, 200);
    pstmt.executeUpdate();

    System.out.println("2件挿入完了");
}
terminal Output 2件挿入完了

4. ResultSet

table ResultSetの主な取得メソッド

メソッド取得できる型
getString(columnName or index)String
getInt(columnName or index)int
getLong(columnName or index)long
getDouble(columnName or index)double
getBoolean(columnName or index)boolean
getDate(columnName or index)java.sql.Date
getObject(columnName or index)Object（任意の型）

⚠️ 試験対策： ResultSetのカーソルは最初「先頭行の前」にあります。next() を1回呼んで初めて最初の行に移動します。カラムインデックスも 1始まりです。

try (Connection conn = DriverManager.getConnection(url, "sa", "");
     Statement stmt = conn.createStatement();
     // ResultSetもtry-with-resourcesでAutoCloseableとして閉じられる
     ResultSet rs = stmt.executeQuery("SELECT * FROM products ORDER BY id")) {

    // rs.next() が false を返すまでループ
    while (rs.next()) {
        // カラム名で取得（推奨）
        int id = rs.getInt("id");
        String name = rs.getString("name");

        // カラムインデックスで取得（1始まり）
        int price = rs.getInt(3);

        System.out.printf("id=%d, name=%s, price=%d%n", id, name, price);
    }
}
terminal Output id=1, name=Coffee, price=500
id=2, name=Tea, price=300
id=3, name=Juice, price=200

5. トランザクション

table トランザクション関連メソッド

メソッド説明
conn.setAutoCommit(false)自動コミットを無効化（トランザクション開始）
conn.commit()変更を確定（DBに永続化）
conn.rollback()変更を全て取り消す
conn.setSavepoint(name)セーブポイントを設定（部分ロールバックが可能）
conn.rollback(savepoint)指定セーブポイントまで戻す

setAutoCommit(false) にすると、SQL実行後に明示的に commit() を呼ぶまでDBへの変更は確定されません。例外発生時は必ず rollback() を呼んでください。

try (Connection conn = DriverManager.getConnection(url, "sa", "")) {
    conn.setAutoCommit(false); // 自動コミットをOFF

    try (Statement stmt = conn.createStatement()) {
        stmt.executeUpdate("UPDATE products SET price = 600 WHERE id = 1");
        stmt.executeUpdate("UPDATE products SET price = 400 WHERE id = 2");

        conn.commit(); // 両方の更新をまとめて確定
        System.out.println("トランザクションコミット成功");

    } catch (SQLException e) {
        conn.rollback(); // どちらかが失敗したら全て取り消し
        System.out.println("ロールバックしました: " + e.getMessage());
    }
}
terminal Output トランザクションコミット成功
