1. オートボクシング / アンボクシング

table プリミティブ型とラッパークラスの比較

プリミティブ型 ラッパークラス 注意点
int Integer 名前が変わる
char Character 名前が変わる
boolean Boolean 頭文字大文字
double Double 頭文字大文字

description 型の自動変換について

Javaでは、基本データ型（intなど）とそれに対応するラッパークラス（Integerなど）の間で、コンパイラが自動的に変換を行います。
基本型からラッパークラスへの変換を「オートボクシング」、逆を「アンボクシング」と呼びます。明示的なキャストなしで代入できるためコードがすっきりします。 

Integer boxed = 10;  // オートボクシング (Integer.valueOf(10) が呼ばれる)
int unboxed = boxed; // アンボクシング (boxed.intValue() が呼ばれる)
System.out.println("boxed: " + boxed + ", unboxed: " + unboxed);
terminal Output boxed: 10, unboxed: 10

table ⚠️ Integer キャッシュの範囲比較

ラッパークラス キャッシュされる範囲
Integer, Byte, Short, Long -128 ～ 127
Character 0 ～ 127
Boolean true / false
Float, Double キャッシュなし

description 同一オブジェクトとして扱われる罠（試験頻出）

ラッパークラスにはメモリを節約するためのキャッシュ機構があります。Integer の場合、-128 ～ 127 の範囲の値は同じインスタンスが再利用されます。
そのため、この範囲内の数値を == で比較すると true になりますが、範囲外だと別インスタンスとして生成されるため false になります。
ラッパークラスの値比較は、必ず equals() を使用してください。 

// キャッシュ範囲内 (-128 ~ 127)
Integer a = 127;
Integer b = 127;
System.out.println("127 == 127: " + (a == b));

// キャッシュ範囲外
Integer c = 128;
Integer d = 128;
System.out.println("128 == 128 (==比較): " + (c == d));
System.out.println("128 == 128 (equals): " + c.equals(d));
terminal Output 127 == 127: true
128 == 128 (==比較): false
128 == 128 (equals): true

2. ジェネリクス

table 型パラメータとワイルドカードの比較

記法 型の状態 読み取り 書き込み
<?> 不明 Object のみ 不可
<T> 確定（呼出時） T として可 T 型なら可
<T extends X> 確定（X 以下） X として可 T 型なら可
<? extends X> 不明（X のサブクラス） X として可 不可
<? super X> 不明（X の親クラス） Object のみ X 型なら可

description ジェネリクスの変性（非変性）と型の制限

型システムには「共変性（サブクラス代入可）」「反変性（スーパークラス代入可）」「非変性（代入不可）」があり、Javaのジェネリクスは「非変性」です。（例: List<Object> に List<String> は代入不可）。
また、総称型クラスのパラメータ <T> はデフォルトで Object 扱いになるため、特定のメソッド（例: doubleValue()）を使いたい場合は <T extends Number> のように型を限定（上限境界）しないとコンパイルエラーになります。

description <> と <?> の違いと null 代入ルール

<>（ダイヤモンド演算子）は、インスタンス生成時に右辺の型推論を行うためのものです。
一方、<?>（非境界ワイルドカード）は変数の型宣言に使用し、「どんな型でも受け入れる」ことを示します。
⚠️ 重要: <?> や <? extends X> で宣言されたリストは実際の型が不明なため、型の安全性が保証できません。そのため、null 以外を add することはできず、コンパイルエラーになります。

// Number のサブクラスに限定するジェネリクスメソッド
public static <T extends Number> double sum(List<T> list) {
    double total = 0;
    for (T item : list) {
        total += item.doubleValue(); // Numberのメソッドが安全に呼べる
    }
    return total;
}

List<Integer> intList = Arrays.asList(1, 2, 3);
System.out.println("Sum: " + sum(intList));
terminal Output Sum: 6.0

3. List<E>

table Listの実装クラス比較

実装クラス 内部構造 インデックス取得 (get) 中間への挿入・削除 特徴
ArrayList 配列 高速 (O(1)) 遅い (O(n)) 最もよく使われる。検索が速い
LinkedList 双方向連結リスト 遅い (O(n)) 高速 (O(1)) 要素の追加・削除が頻繁な場合に有利
Vector 配列 高速 (O(1)) 遅い (O(n)) スレッドセーフ（同期化されている）。現在はほぼ使われない

description 順序付きリストの基本

List は「要素の追加順序を保持し、重複を許可する」コレクションです。最も代表的な ArrayList は内部的に配列を使用しているため、インデックスを指定した要素の取得が非常に高速です。

List<String> list = new ArrayList<>();
list.add("Cherry");
list.add("Apple");
list.add("Banana");
list.add("Apple"); // 重複を許可

System.out.println("ArrayList: " + list);
System.out.println("get(1): " + list.get(1));
terminal Output ArrayList: [Cherry, Apple, Banana, Apple]
get(1): Apple

table ⚠️ 不変リスト生成の罠（試験頻出）

生成方法 変更可否 null要素 特徴
new ArrayList<>() 全て可 可 通常の可変リスト
Arrays.asList(...) setのみ可 可 固定サイズ。add/remove は例外
List.of(...) 全て不可 不可 Java 9以降。完全不変

description 生成方法による可変性の違い

Listを生成する際、使用するメソッドによってその後の「変更可否（可変性）」が異なります。
特に Arrays.asList() は、要素の変更（set）はできますが、追加や削除を行おうとすると例外（UnsupportedOperationException）が発生します。
また、Java 9 で導入された List.of() は、要素の変更も追加・削除も一切できない「完全な不変リスト」を作成します。

// new ArrayList<>(): 通常の可変リスト（最もよく使われる）
List<String> arrayList = new ArrayList<>();
arrayList.add("Cherry"); arrayList.add("Apple"); arrayList.add("Banana"); arrayList.add("Apple");
System.out.println("ArrayList: " + arrayList); // [Cherry, Apple, Banana, Apple]

// Arrays.asList(): 固定サイズ。setは可、add/removeは例外
List<String> asListEx = Arrays.asList("X", "Y", "Z");
asListEx.set(0, "A"); // OK
System.out.println("asList after set: " + asListEx); // [A, Y, Z]
try { asListEx.add("W"); } catch (UnsupportedOperationException e) {
    System.out.println("asList.add: " + e.getClass().getSimpleName()); // UnsupportedOperationException
}

// LinkedList: 中間挿入・削除が高速 (O(1))
List<String> linked = new LinkedList<>(Arrays.asList("A", "B", "C"));
linked.add(1, "X"); // index=1に挿入
System.out.println("LinkedList: " + linked); // [A, X, B, C]

// List.of(): Java9+. 完全不変（add/set/remove全て例外）
List<String> immutableList = List.of("X", "Y", "Z");
System.out.println("List.of(): " + immutableList);
terminal Output ArrayList: [Cherry, Apple, Banana, Apple]
asList after set: [A, Y, Z]
asList.add: UnsupportedOperationException
LinkedList: [A, X, B, C]
List.of(): [X, Y, Z]

4. Set<E>

table Setの実装クラス比較

実装クラス 順序の保証 検索速度 主な用途
HashSet なし 最高速 (O(1)) 順序を気にせず重複を省きたい時
LinkedHashSet 挿入順 高速 (O(1)) 入れた順番通りに取り出したい時
TreeSet 自然順序（昇順など） やや遅い (O(log n)) 常にソートされた状態を保ちたい時

description 重複不可コレクションの使い分け

Set は「重複する要素を持たない」コレクションです。
HashSet は最も一般的で処理が高速ですが、順序は全く保証されません。順序が必要な場合は LinkedHashSet（挿入順）や TreeSet（ソート順）を使用します。 

Set<String> hashSet = new HashSet<>(Arrays.asList("Dog", "Cat", "Bird", "Cat"));
System.out.println("HashSet (順序保証なし): " + hashSet);

Set<String> linkedHashSet = new LinkedHashSet<>(Arrays.asList("Dog", "Cat", "Bird", "Cat"));
System.out.println("LinkedHashSet (挿入順): " + linkedHashSet);

Set<String> treeSet = new TreeSet<>(Arrays.asList("Dog", "Cat", "Bird"));
System.out.println("TreeSet (ソート順): " + treeSet);
terminal Output HashSet (順序保証なし): [Cat, Bird, Dog]
LinkedHashSet (挿入順): [Dog, Cat, Bird]
TreeSet (ソート順): [Bird, Cat, Dog]

5. Map<K,V>

table Mapの実装クラス比較

実装クラス キーの順序 nullキーの許可 特徴
HashMap 保証なし 可 最もよく使われる。検索が速い
LinkedHashMap 挿入順 可 挿入された順番を保持する
TreeMap キーの昇順 不可 常にキーでソートされる

description キーと値のペアでのデータ管理

Map はキー（Key）と値（Value）のペアでデータを管理します。キーの重複は許可されません。
もし既に存在するキーに対して再度 put メソッドを実行した場合、古い値が新しい値で上書きされます。

Map<Integer, String> map = new HashMap<>();
map.put(3, "Three");
map.put(1, "One");
map.put(2, "Two");

// キー「1」は既に存在するため、「One_Updated」に上書きされる
map.put(1, "One_Updated"); 

System.out.println("HashMap: " + map);
terminal Output HashMap: {1=One_Updated, 2=Two, 3=Three}

description 便利な値の取得・追加メソッド

getOrDefault は、指定したキーが存在しない場合に例外やnullを返す代わりに、あらかじめ指定したデフォルト値を返します。
putIfAbsent は、「もしそのキーが存在していなければ追加する」という処理を行います。すでにキーが存在する場合は何も上書きされません。 

// キー99 は存在しないため、第二引数の "未登録" が返る
System.out.println("getOrDefault(99): " + map.getOrDefault(99, "未登録"));

// キー1 は既に存在するため、この putIfAbsent は無視（上書きされない）
map.putIfAbsent(1, "Won't overwrite");
System.out.println("putIfAbsent(1)後の値: " + map.get(1));
terminal Output getOrDefault(99): 未登録
putIfAbsent(1)後の値: One_Updated

description Map.Entry と 不変Mapの初期化

Mapに格納されているキーと値のペア（エントリー）を取得するには entrySet() メソッドを使用します。戻り値の型は Set<Map.Entry<K, V>> となります。
また、Java 9以降では Map.entry(K, V) という static メソッドを使ってエントリーを生成し、Map.ofEntries() で初期化することができます（インスタンス化は不要ですが、変更不可のMapになります）。
受け取る際は var（ローカル変数型推論）を使うと、Map.Entry<Integer, String> のような長い型名を書かずにスッキリ記述できます。

// Map.ofEntries と Map.entry は staticメソッドなのでインスタンス不要。変更不可なMapを生成。
var immutableMap = Map.ofEntries(
    Map.entry(1, "Apple"),
    Map.entry(2, "Banana")
);

// entrySet() で Map.Entry の Set を取得し、ループで回す
for (var entry : immutableMap.entrySet()) {
    System.out.println("Key: " + entry.getKey() + ", Value: " + entry.getValue());
}
terminal Output Key: 1, Value: Apple
Key: 2, Value: Banana

6. Queue / Deque

table Queue / Deque メソッドの比較（例外スロー vs 特殊値返却）

操作 例外スロー 特殊値返却 (推奨) Deque専用 (両端操作)
末尾に追加 add(e) offer(e) offerLast(e)
先頭に追加 - - offerFirst(e)
先頭を取得＆削除 remove() poll() pollFirst()
末尾を取得＆削除 - - pollLast()
先頭を参照のみ element() peek() peekFirst()
末尾を参照のみ - - peekLast()

description Queue (先入れ先出し) と Deque (両端キュー)

Queue は先入れ先出し（FIFO）のデータ構造です。
一方、Deque (Double Ended Queue) は、先頭と末尾の両方から要素の追加・取得・削除ができるデータ構造です（スタックとしても使用可能）。
操作時には例外をスローしない offer() / poll() / peek() や、Deque専用の offerFirst() / pollLast() 等を使うのが安全です（これらはエラー時にプログラムをクラッシュさせず、falseやnullを返します）。

// Queue: FIFO操作 (例外スロー vs 特殊値返却)
Queue<String> queue = new LinkedList<>();
queue.add("First");    // 失敗時 IllegalStateException
queue.offer("Second"); // 失敗時 false返却（推奨）
queue.offer("Third");

System.out.println("element() 先頭参照:  " + queue.element()); // 空なら NoSuchElementException
System.out.println("peek()    先頭参照:  " + queue.peek());    // 空なら null
System.out.println("remove()  先頭取得&削除: " + queue.remove()); // 空なら NoSuchElementException
System.out.println("poll()    先頭取得&削除: " + queue.poll());   // 空なら null
System.out.println("残り: " + queue);

// Deque: 両端キュー（先頭・末尾両方から操作可能）
Deque<String> deque = new ArrayDeque<>();
deque.offerFirst("B");  // 先頭に追加
deque.offerLast("C");   // 末尾に追加
deque.offerFirst("A");  // 先頭に追加 → [A, B, C]
System.out.println("deque: " + deque);
System.out.println("peekFirst(): " + deque.peekFirst()); // A
System.out.println("peekLast():  " + deque.peekLast());  // C
System.out.println("pollFirst(): " + deque.pollFirst()); // A
System.out.println("pollLast():  " + deque.pollLast());  // C
System.out.println("残り: " + deque); // [B]
terminal Output element() 先頭参照: First
peek() 先頭参照: First
remove() 先頭取得&削除: First
poll() 先頭取得&削除: Second
残り: [Third]
deque: [A, B, C]
peekFirst(): A
peekLast(): C
pollFirst(): A
pollLast(): C
残り: [B]

7. Comparator

table Comparable と Comparator の比較

Comparable Comparator
所属パッケージ java.lang java.util
実装メソッド compareTo(T o) compare(T o1, T o2)
実装場所 比較対象のクラス自身 別クラス、またはラムダ式
用途 自然順序（デフォルトの並び順） カスタム順序（複数定義可能）

description 柔軟なソート条件の指定

要素を並べ替える際、Comparator インターフェースを使用することで独自のソート順を定義できます。
標準で用意されている Comparator.naturalOrder() を指定すれば昇順（自然順序）に、Comparator.reverseOrder() を指定すれば降順（逆順）にソートされます。

List<String> names = new ArrayList<>(Arrays.asList("Charlie", "Alice", "Bob"));

names.sort(Comparator.naturalOrder()); // 昇順
System.out.println("naturalOrder: " + names);

names.sort(Comparator.reverseOrder()); // 降順
System.out.println("reverseOrder: " + names);
terminal Output naturalOrder: [Alice, Bob, Charlie]
reverseOrder: [Charlie, Bob, Alice]

8. Collections ユーティリティ

table 主な Collections のメソッド一覧

メソッド 説明
Collections.sort(list) リストを昇順にソートする（Comparable実装が必要）
Collections.reverse(list) リストの順序を逆にする
Collections.max / min コレクション内の最大値 / 最小値を返す
Collections.frequency(col, obj) コレクション内に指定した要素がいくつあるか数える

description コレクション操作を補助する静的メソッド群

Collections クラス（末尾に 's' がつく）は、コレクションの操作に便利な static メソッドを集めたユーティリティクラスです。
インスタンス化せずにそのまま Collections.メソッド名 と呼び出すことで、並べ替えや検索を簡単に行うことができます。

List<Integer> nums = new ArrayList<>(Arrays.asList(3, 1, 4, 1, 5, 9));

System.out.println("max: " + Collections.max(nums));
System.out.println("min: " + Collections.min(nums));
System.out.println("frequency of 1 (1の出現回数): " + Collections.frequency(nums, 1));

Collections.sort(nums); // リスト自身を昇順に書き換える
System.out.println("sorted:   " + nums);

// reverse(): リストの順序を逆にする
Collections.reverse(nums);
System.out.println("reversed: " + nums);
terminal Output max: 9
min: 1
frequency of 1 (1の出現回数): 2
sorted: [1, 1, 3, 4, 5, 9]
reversed: [9, 5, 4, 3, 1, 1]
