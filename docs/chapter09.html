2. try-catch-finally の実行順序

⚠️ 試験頻出： finally は return や例外が発生しても必ず実行されます。また、try 内で return があっても finally が先に実行されてから戻ります。finally 内で return すると、try 内の return を上書きします。

public static int tryCatchFinallyDemo() {
    try {
        System.out.println("try実行");
        return 1; // finallyが先に実行される
    } catch (Exception e) {
        System.out.println("catch実行");
        return 2;
    } finally {
        System.out.println("finally実行 (必ず実行)");
        // ここにreturnを書くとtryのreturnを上書きする（試験頻出の罠）
    }
}

// 複数のcatch: 子クラスを先に書く（親クラスを先に書くとコンパイルエラー）
try {
    String s = null;
    s.length(); // NullPointerException
} catch (NullPointerException e) {
    System.out.println("NPE caught");        // ← 先にこちらがキャッチ
} catch (RuntimeException e) {
    System.out.println("RuntimeEx caught");  // ← 親クラスは後
}

System.out.println("戻り値: " + tryCatchFinallyDemo());
terminal Output NPE caught
try実行
finally実行 (必ず実行)
戻り値: 1

3. マルチキャッチと再スロー

table マルチキャッチのルール

ルール説明
構文catch (ExceptionA | ExceptionB e)
変数 e の型列挙した例外の共通の親クラス（暗黙的に final）
再代入不可（eは暗黙的finalなので、ブロック内で e = new ... はコンパイルエラー）
継承関係のある例外の併記不可（コンパイルエラー）
// マルチキャッチ: 処理が同じなら1つにまとめられる
try {
    if (Math.random() > 0.5)
        throw new NumberFormatException("数値エラー");
    else
        throw new IllegalArgumentException("引数エラー");
} catch (NumberFormatException | IllegalArgumentException e) {
    // e はここでは実質的にfinal（再代入不可）
    System.out.println("どちらかの例外: " + e.getMessage());
}

// 再スロー（rethrow）: 型を絞って再スローできる
void process(boolean flag) throws IOException {
    try {
        if (flag) throw new IOException("IOエラー");
    } catch (Exception e) {
        System.out.println("ログ記録: " + e.getMessage());
        throw e; // Java 7以降: 実際にスローされたより具体的な型で再スロー可能
    }
}
terminal Output どちらかの例外: 数値エラー (または 引数エラー)

4. カスタム例外クラスの作成

// 検査例外として定義: Exceptionを継承
class InsufficientFundsException extends Exception {
    private final int shortage;

    public InsufficientFundsException(String message, int shortage) {
        super(message);          // 親クラスにメッセージを渡す
        this.shortage = shortage;
    }

    // 原因例外を渡すコンストラクタ（例外チェーン）
    public InsufficientFundsException(String message, Throwable cause) {
        super(message, cause);
    }

    public int getShortage() { return shortage; }
}

// 非検査例外として定義: RuntimeExceptionを継承
class InvalidUserException extends RuntimeException {
    public InvalidUserException(String message) { super(message); }
}

// 使用例
void withdraw(int balance, int amount) throws InsufficientFundsException {
    if (amount > balance) {
        throw new InsufficientFundsException(
            "残高不足", amount - balance
        );
    }
}

try {
    withdraw(1000, 1500);
} catch (InsufficientFundsException e) {
    System.out.println(e.getMessage() + " (" + e.getShortage() + "円不足)");
}
terminal Output 残高不足 (500円不足)

5. アサーション (assert)

table assert文のルール

項目内容
構文1assert 条件式;
構文2assert 条件式 : メッセージ式;
有効化JVM起動時に -ea または -enableassertions オプションが必要
失敗時AssertionError がスロー（Errorなのでcatchすべきでない）
無効化（デフォルト）-da または オプションなし。本番環境では無効が一般的
禁止ケース副作用を含む式をassertに使ってはいけない（有効/無効で動作が変わる）
// assert文の例
int score = 85;
assert score >= 0 && score <= 100 : "スコアは0-100の範囲外: " + score;
System.out.println("スコア: " + score);

// NG例: 副作用を含む式（有効/無効で挙動が変わる）
int count = 0;
assert ++count > 0; // -ea無効時はcountが加算されない → バグの原因

// 実行方法
// java -ea MyClass   → assertが有効（開発・テスト時）
// java MyClass       → assertが無効（本番時のデフォルト）
terminal Output スコア: 85
※ -ea オプションなしの場合、assert文はスキップされる
