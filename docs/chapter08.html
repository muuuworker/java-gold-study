1. 組み込みアノテーション

table 主要な組み込みアノテーション

アノテーション用途効果
@Overrideメソッドに付与スーパークラス/インタフェースのメソッドをオーバーライドしていることを宣言。していない場合はコンパイルエラー
@Deprecatedクラス/メソッド等に付与非推奨であることを示す。使用時にコンパイラが警告を出す
@SuppressWarningsクラス/メソッド等に付与指定した種類のコンパイラ警告を抑制する
@FunctionalInterfaceインタフェースに付与関数型インタフェース（SAM）であることを宣言。違反時にコンパイルエラー
@SafeVarargs可変長引数メソッドに付与ジェネリクスの可変長引数に関する警告を抑制
// @Override: タイポ防止に有効
class Animal {
    public String sound() { return "..."; }
}
class Dog extends Animal {
    @Override
    public String sound() { return "Woof"; } // スペルミスがあればコンパイルエラー
}

// @Deprecated: 非推奨メソッドのマーキング
class OldApi {
    @Deprecated(since = "2.0", forRemoval = true)
    public void oldMethod() { System.out.println("旧メソッド"); }
}

// @SuppressWarnings: 特定の警告を抑制
@SuppressWarnings("unchecked") // ジェネリクス関連の未チェック警告を抑制
public void rawTypeMethod() {
    List rawList = new ArrayList(); // 通常は警告が出るが抑制される
    rawList.add("test");
}
terminal Output コンパイル時に検証・警告制御が行われる（実行時の出力はなし）

2. メタアノテーション

カスタムアノテーションを定義する際に、そのアノテーション自身の挙動を制御するために使います。

table @Retention ポリシーの比較（試験頻出）

RetentionPolicy保持場所リフレクションで取得主な用途
SOURCEソースコードのみ不可コンパイル時のヒント（@Override等）
CLASSclassファイルまで（デフォルト）不可バイトコード解析ツール
RUNTIMEJVM実行中も保持可フレームワーク（Spring等）でのDI/AOP

table @Target の主な対象（ElementType）

ElementType付与できる場所
TYPEクラス、インタフェース、enum、アノテーション型
METHODメソッド
FIELDフィールド（インスタンス変数）
CONSTRUCTORコンストラクタ
PARAMETERメソッドの引数
LOCAL_VARIABLEローカル変数
ANNOTATION_TYPEアノテーション型（メタアノテーション用）

3. カスタムアノテーションの定義

description 定義ルール

① @interface キーワードで定義する。
② 要素（属性）は引数なしのメソッドとして宣言する。
③ default でデフォルト値を指定できる。
④ 要素が value 1つだけなら、使用時に 要素名= を省略できる。
⑤ 要素には配列型も使える。 

import java.lang.annotation.*;

// カスタムアノテーションの定義
@Retention(RetentionPolicy.RUNTIME) // リフレクションで取得可能にする
@Target(ElementType.METHOD)         // メソッドにのみ付与可能
public @interface TestCase {
    String value() default "テスト"; // "value"という要素名なら省略可
    int priority() default 1;
    String[] tags() default {};      // 配列型の要素
}

// 使用例
public class MyTests {
    @TestCase("ログインテスト")               // value=省略, priority=1(デフォルト)
    public void testLogin() { }

    @TestCase(value = "決済テスト", priority = 10, tags = {"payment", "critical"})
    public void testPayment() { }
}
terminal Output 定義のみ（実行時の出力はなし）

4. リフレクションによるアノテーションの取得

RetentionPolicy.RUNTIMEのアノテーションは、実行時にリフレクションAPIを使って取得・処理できます。フレームワークの内部実装で多用される仕組みです。

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface TestMethod {
    String value() default "Default Test";
}

class MyService {
    @TestMethod("ユーザー作成テスト")
    public void createUser() { System.out.println("User created."); }

    public void deleteUser() { System.out.println("User deleted."); }
}

// リフレクションでアノテーション付きメソッドを動的に実行
Class<?> clazz = MyService.class;
MyService service = new MyService();

for (var method : clazz.getMethods()) {
    if (method.isAnnotationPresent(TestMethod.class)) {
        TestMethod annotation = method.getAnnotation(TestMethod.class);
        System.out.println("実行テスト: " + annotation.value());
        method.invoke(service); // メソッドを動的呼び出し
    }
}
terminal Output 実行テスト: ユーザー作成テスト
User created.
