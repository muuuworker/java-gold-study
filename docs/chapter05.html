1. Path と Paths

table 旧来のFileクラス vs NIO.2のPathの比較

java.io.File（旧来）java.nio.file.Path（NIO.2）
生成方法new File("path")Paths.get("path") または Path.of("path")
エラー処理多くのメソッドがbooleanを返す（例外なし）失敗時に例外をスロー（より安全）
ファイル操作Fileクラス自身Filesクラスのstaticメソッドを使用
シンボリックリンク非対応対応
// Path.of() はJava11以降 (Paths.get()と等価)
Path p1 = Path.of("dir", "file.txt");      // dir/file.txt
Path p2 = Paths.get("/home/user/doc.txt");

// 便利なPathメソッド
System.out.println(p2.getFileName());  // doc.txt
System.out.println(p2.getParent());    // /home/user
System.out.println(p2.getRoot());      // /

// resolve: パスの結合
Path base = Path.of("/home/user");
Path resolved = base.resolve("docs/readme.txt");
System.out.println(resolved); // /home/user/docs/readme.txt

// relativize: 相対パスを取得
Path other = Path.of("/home/user/docs/readme.txt");
System.out.println(base.relativize(other)); // docs/readme.txt

Path file = Path.of("sample.txt");
Path dir  = Path.of("mydir");
Path sub  = Path.of("mydir/sub/deep");

// exists: ファイル/ディレクトリの存在確認
System.out.println("exists(before): " + Files.exists(file)); // false

// createFile: 新規ファイル作成
Files.createFile(file);
System.out.println("exists(after):  " + Files.exists(file)); // true

// createDirectory: 1階層のみ作成（中間ディレクトリがないと例外）
Files.createDirectory(dir);

// createDirectories: 中間ディレクトリも含めて全て作成
Files.createDirectories(sub);

// writeString / readString（Java 11+）
Files.writeString(file, "Hello NIO.2!\nSecond Line\n");
String content = Files.readString(file);
System.out.println("readString:\n" + content);

// readAllLines: 全行をList<String>で読む
List<String> lines = Files.readAllLines(file);
System.out.println("readAllLines: " + lines);

// write: List<String>をファイルに書く
Files.write(Path.of("lines.txt"), lines);

// isDirectory: ディレクトリかどうか
System.out.println("isDirectory(dir):  " + Files.isDirectory(dir));  // true
System.out.println("isDirectory(file): " + Files.isDirectory(file)); // false

// size: ファイルサイズ（バイト）
System.out.println("size: " + Files.size(file) + " bytes");

// copy: コピー（REPLACE_EXISTINGで上書き許可）
Files.copy(file, Path.of("backup.txt"), StandardCopyOption.REPLACE_EXISTING);

// move: 移動/リネーム
Files.move(Path.of("backup.txt"), Path.of("renamed.txt"), StandardCopyOption.REPLACE_EXISTING);

// delete: 削除（存在しない場合は NoSuchFileException）
Files.delete(Path.of("renamed.txt"));

// deleteIfExists: 存在する場合のみ削除（存在しなくても例外なし）
Files.deleteIfExists(file);
Files.deleteIfExists(Path.of("lines.txt"));

3. StreamによるファイルI/O

Files.lines() を使うと、ファイルの各行をストリームとして処理できます。大きなファイルでも全行をメモリに読み込まないため、効率的です。必ずtry-with-resourcesで閉じてください。

Path logFile = Path.of("app.log");
Files.writeString(logFile, "INFO: start\nERROR: failed\nINFO: end\n");

// Files.lines(): 遅延評価でファイルを1行ずつStream処理
try (Stream<String> stream = Files.lines(logFile)) {
    long errorCount = stream
        .filter(line -> line.startsWith("ERROR"))
        .peek(line -> System.out.println("Found: " + line))
        .count();
    System.out.println("エラー件数: " + errorCount);
}

Files.deleteIfExists(logFile);
terminal Output Found: ERROR: failed
エラー件数: 1

4. try-with-resources

table AutoCloseableとCloseableの違い

インタフェースメソッドスローできる例外主な用途
AutoCloseableclose() throws Exception検査例外・非検査例外try-with-resourcesの対象基底
Closeableclose() throws IOExceptionIOExceptionIOストリーム系クラス

⚠️ 試験対策： try-with-resourcesブロックは、finallyより先にclose()が呼ばれます。また、複数リソースのclose()は宣言の逆順で呼ばれます。

// try-with-resources: ブロックを抜けると自動的にclose()される
Path file = Path.of("resource_test.txt");
Files.writeString(file, "test content");

// BufferedReaderはCloseableを実装しているのでtry-with-resourcesで使える
try (BufferedReader reader = Files.newBufferedReader(file)) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} // ここでreader.close()が自動的に呼ばれる
// catch, finallyは省略可能

// 複数リソース: セミコロンで区切る。closeは宣言の逆順（res2→res1）
try (var res1 = Files.newBufferedReader(file);
     var res2 = Files.newBufferedWriter(Path.of("out.txt"))) {
    res2.write(res1.readLine());
}

Files.deleteIfExists(file);
Files.deleteIfExists(Path.of("out.txt"));
terminal Output test content

5. ディレクトリ走査

table ディレクトリ走査メソッドの比較

メソッド深さ戻り値説明
Files.list(path)1階層のみStream<Path>直下の要素を列挙（再帰なし）
Files.walk(path)全階層Stream<Path>再帰的に全要素を列挙
Files.find(path, depth, matcher)指定深さStream<Path>条件付きで検索
// Files.walk: 指定ディレクトリ以下を再帰的に全列挙
Path dir = Path.of("."); // カレントディレクトリ

try (Stream<Path> paths = Files.walk(dir, 1)) { // depth=1: 直下のみ
    paths.filter(Files::isRegularFile)            // ファイルのみ
         .filter(p -> p.toString().endsWith(".java"))
         .forEach(System.out::println);
}

// Files.find: 属性情報(BasicFileAttributes)を使った高度な検索
try (Stream<Path> found = Files.find(dir, 2,
        (p, attr) -> attr.isRegularFile() && attr.size() > 100)) {
    found.forEach(System.out::println);
}
